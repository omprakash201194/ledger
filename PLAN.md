# Ledger — Implementation Plan

Family Financial Digital Legacy Register. A self-hosted, privacy-first webapp that documents the complete financial picture of a household so trusted persons can act immediately in the event of death or incapacity.

---

## Decisions & Rationale

| Decision | Choice | Reason |
|---|---|---|
| Auth | Spring Security OAuth2 (Google) + JWT | Same pattern as SpendStack — no Firebase. Financial data stays on-homelab. |
| Database | Shared homelab Postgres | Relational model fits perfectly; no NoSQL needed. Redis already covers caching. |
| Credential vault | Not built | Out of scope. `DigitalAccount` stores only where credentials live, never the credentials themselves. |
| Terminology | "Trusted person" instead of "nominee" | More human, less legal-form language. |
| Reminders | In-app alerts only (MVP) | No email plumbing needed for MVP. Spring `@Scheduled` scans due dates nightly. |
| Public exposure | Cloudflare Tunnel | Cloudflared already running in homelab. Add routing rule in Cloudflare dashboard only. |
| NoSQL | Not added | Postgres JSONB covers semi-structured needs. Redis already covers caching. Limited RAM on i3-4150. |

## Domain Model

All entities have a `user_id` FK (one principal per account, multi-user = household).

### User
The principal who owns the register.
- `id`, `email`, `name`, `provider` (local/google), `password_hash`
- `date_of_birth`, `pan_number` (masked), `aadhaar_ref` (reference/location only)

### TrustedPerson
People the family must contact in an emergency. Replaces "nominee" throughout.
- `id`, `user_id`, `name`, `relationship`, `phone`, `email`, `type` (FAMILY | ADVISOR | EXECUTOR)
- `notes`

### Asset
Every owned asset — financial and physical.
- `id`, `user_id`, `asset_type` (enum), `description`, `institution`, `account_number` (masked)
- `holding_mode` (SINGLE | JOINT | EITHER_OR_SURVIVOR), `joint_holder_name`
- `trusted_person_id` (registered trusted person), `approx_value`, `value_as_of`
- `document_location`, `remarks`
- `document_location`, `remarks`
- `maturity_date` (for FD, RD, NPS — used by alert scanner)
- **Asset type enum:** SAVINGS_ACCOUNT, CURRENT_ACCOUNT, FIXED_DEPOSIT, RECURRING_DEPOSIT, PPF, EPF, NPS, MUTUAL_FUND, EQUITY, GOLD, REAL_ESTATE, VEHICLE, BANK_LOCKER, OTHER

### Liability
Outstanding debts and obligations.
- `id`, `user_id`, `liability_type` (enum), `lender`, `account_number` (masked)
- `original_amount`, `outstanding_balance`, `emi_amount`, `tenure_end_date`
- `linked_asset_id` (FK to Asset, nullable), `remarks`
- **Liability type enum:** HOME_LOAN, CAR_LOAN, PERSONAL_LOAN, EDUCATION_LOAN, CREDIT_CARD, OTHER

### InsurancePolicy
Life, health, and general insurance.
- `id`, `user_id`, `policy_type` (enum), `insurer`, `policy_number` (masked)
- `life_assured`, `sum_assured`, `premium_amount`, `premium_due_date` (day-of-month + month)
- `trusted_person_id` (beneficiary), `maturity_date`, `document_location`, `remarks`
- **Policy type enum:** TERM_LIFE, WHOLE_LIFE, HEALTH, VEHICLE, PROPERTY, OTHER

### DigitalAccount
Online accounts with location of credentials — never the credentials themselves.
- `id`, `user_id`, `category` (enum), `service_name`, `username`
- `credential_location` (e.g. "In Bitwarden vault"), `two_fa_method`, `recovery_contact`
- `action_on_death` (TRANSFER | ARCHIVE | CLOSE | MEMORIALIZE), `remarks`
- **Category enum:** PASSWORD_MANAGER, EMAIL, BANKING, INVESTMENT, SOCIAL_MEDIA, GOVERNMENT, SUBSCRIPTION, OTHER

### RecurringObligation
Auto-debits and standing instructions that keep running until stopped.
- `id`, `user_id`, `obligation_type` (enum), `payee`, `amount`, `frequency` (enum)
- `due_day` (1–31), `payment_source` (account/card description), `action_on_death` (CONTINUE | CANCEL | REVIEW)
- `remarks`
- **Obligation type enum:** LOAN_EMI, SIP, INSURANCE_PREMIUM, SUBSCRIPTION, UTILITY, OTHER
- **Frequency enum:** MONTHLY, QUARTERLY, HALF_YEARLY, YEARLY

### WillRecord
Metadata and location of the Will — the app never stores Will text.
- `id`, `user_id`, `has_will`, `will_type` (SINGLE | JOINT | NONE)
- `location` (physical location of the original), `executor_id` (FK TrustedPerson)
- `registered_with` (court/notary/registrar), `review_reminder_date`, `notes`

### Alert (in-app only)
Generated by the nightly scheduler. Read-only from the frontend.
- `id`, `user_id`, `alert_type` (enum), `title`, `message`, `source_entity` (e.g. "policy_id:42")
- `due_date`, `is_read`, `created_at`
- **Alert type enum:** INSURANCE_PREMIUM_DUE, EMI_DUE, WILL_REVIEW_DUE, WILL_NO_REVIEW, OBLIGATION_REVIEW, ASSET_VALUE_STALE, NOMINEE_MISSING, FD_MATURITY_DUE, EMI_ENDING_SOON

**Profiles:**
- `local` — Postgres on `localhost:5432`, CORS open to `localhost:5173`
- `k8s` — Postgres on `postgres.homelab.svc.cluster.local:5432`

**Alert scheduler:** runs nightly at 02:30 via `@Scheduled(cron = "0 30 2 * * *")`. Scans:
- InsurancePolicy where `premium_due_date` is within 30 days → `INSURANCE_PREMIUM_DUE`
- RecurringObligation where `due_day` is within 7 days (monthly) → `EMI_DUE`
- WillRecord where `review_reminder_date` is within 30 days → `WILL_REVIEW_DUE`
- WillRecord where `has_will=true` and last updated > 18 months ago, no active reminder → `WILL_NO_REVIEW`
- Asset where `value_as_of` is older than 90 days → `ASSET_VALUE_STALE`
- Asset where `trusted_person_id` is null (aggregate per user) → `NOMINEE_MISSING`
- Asset (FD/RD/NPS) where `maturity_date` is within 30 days → `FD_MATURITY_DUE`
- Liability where `tenure_end_date` is within 60 days → `EMI_ENDING_SOON`

- `DB_PASSWORD` — reuses existing `postgres-secret` in homelab namespace

## Cloudflare Tunnel Exposure

No code changes required. Cloudflared already runs in the `homelab` namespace with a token-based config. Routing lives entirely in the Cloudflare dashboard.

**Steps (done once the app is deployed locally):**
1. Log into Cloudflare Zero Trust → Networks → Tunnels → select your tunnel
2. Public Hostnames → Add hostname:
   - **Subdomain / domain:** `ledger.<yourdomain>.com`
   - **Service type:** HTTPS
   - **URL:** `ledger.homelab.local`
   - **TLS → No TLS Verify:** enabled (internal cert is self-signed)
3. DNS record is created automatically by Cloudflare

The nginx ingress handles the internal TLS. Cloudflare terminates the public-facing TLS with their managed cert.
